# RT 使用時の背景が標準レンダラより明るくなる原因

## 結論（想定される主因）

1. **カラースペースの不一致**（最有力）
2. **背景の「中身」の違い**（スカイボックス vs 単色）

---

## 1. カラースペース（sRGB → リニア）

### 現象
- RT の Miss シェーダでは `_CameraBackgroundColor`（= `Camera.backgroundColor`）をそのまま使っている。
- 出力先は **R16G16B16A16_SFloat**（リニアの Half Float）。

### 原因
- Unity の `Camera.backgroundColor` はインスペクタで選んだ色＝**sRGB で保持**されている。
- プロジェクトが **Linear** のとき、この sRGB 値をリニアバッファにそのまま書き込むと、「リニアとして解釈された明るい値」になる。
- 例: sRGB の 0.5 はリニアでは約 0.21。0.5 をリニアとして表示すると約 2.2 倍明るく見える。

### 対応
- **Linear プロジェクト**のときだけ、C# 側で `Camera.backgroundColor` をリニアに変換してからシェーダに渡す。
  - 例: `QualitySettings.activeColorSpace == ColorSpace.Linear` なら `cam.backgroundColor.linear` を渡す。

---

## 2. 背景の「中身」の違い（スカイボックス vs 単色）

### 現象
- 標準 URP: **Clear Flags = Skybox** ならスカイボックスを描画（グラデーション・プロシージャルなど）。
- RT パス: フレーム全体を上書きするため、ヒットしなかったピクセルは **Miss で単色**（`_CameraBackgroundColor`）になる。

### 原因
- スカイボックスは多くの場合、空の上半分が明るく、地平付近が暗いなど **局所的に暗い**。
- RT ではその代わりに **カメラの単色**が全面に出るため、同じカメラ背景色でも「全体的に明るく」見えやすい。

### 対応（必要に応じて）
- 見た目を揃えたい場合は、Miss シェーダでスカイボックスをサンプリングする実装を検討する（レイ方向からスカイ用テクスチャ/プロシージャルを参照する等）。
- 単色でよい場合は、上記のリニア変換だけでかなり改善する。
