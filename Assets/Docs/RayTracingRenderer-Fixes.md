# レイトレ表示まわりで直したこと（まとめ）

## 1. 青一色（全レイが miss）

**原因**
- **インスタンスマスク**: レイヤー未指定時に `0xFF` を渡していた。TraceRay のマスクとインスタンス側のマスクの解釈で、ヒットしなくなっていた可能性あり。
- **レイ方向**: ビュー空間の前方向を誤ると、レイがジオメトリと交差しない。

**対応**
- レイヤーマスク 0 のときは C# で `0xFF` を渡して「全インスタンスにヒット対象」にした（シェーダ側で `_InstanceMask` が 0 のときは `0xFFFFFFFF` を用いるフォールバックあり）。
- ビュー空間の前方向は **-Z**（カメラから -Z 方向を見る）に統一。

---

## 2. 向きがおかしい（カメラが -Z を見ている）

**原因**
- 一時的にビュー空間の前を +Z にしていたが、Perspective RT カメラは「原点から -Z 方向」を見る前提だった。

**対応**
- ビュー空間の前方向を **-Z** に戻した（`viewDir` の Z を `-1.0`、オルトのレイ方向も `(0,0,-1)`）。

---

## 3. 原点付近からレイが出ている（カメラを動かしても視点が原点のまま）

**原因**
- パースペクティブのレイ始点に `_CameraToWorldMatrix[3].xyz`（行列の平行移動成分）を使っていた。
- `SetRayTracingMatrixParam` で渡した行列のレイアウト（列優先／行優先）の違いで、`[3].xyz` が正しくカメラ位置にならず、常に (0,0,0) 付近になっていた可能性が高い。

**対応**
- カメラ位置を **別パラメータ** で渡すように変更。
  - C#: `_cmd.SetRayTracingVectorParam(..., "_CameraPosition", cam.transform.position)`
  - シェーダー: パース用の `rayOrigin` に `_CameraPosition` を使用。

---

## 4. ガンマ問題の解消（OutputTexture を Linear Float に）

**現象**: RT レンダリングが標準レンダリングより明るく／彩度が強く見えた。

**原因の考察**: OutputTexture を **R8G8B8A8_SRGB** にしていた場合、シェーダーから書き込んだリニア値は GPU により sRGB に変換されてテクスチャに格納される。そのテクスチャを **GUI（OnGUI の DrawTexture や RawImage 等）で表示する際に、Unity が sRGB テクスチャとしてさらにガンマ補正をかけてしまう**可能性があり、結果として二重補正で見た目がおかしくなっていた。

**対応**: OutputTexture のフォーマットを **R16G16B16A16_SFloat**（リニア・Half Float）に変更。シェーダーはリニアのまま書き、テクスチャにもリニアで保存されるため、GUI 表示で余計なガンマ補正がかからず、標準レンダリングと見た目が揃った。

---

## まとめ

| 現象 | 原因 | 修正 |
|------|------|------|
| 青一色 | インスタンスマスク・レイ方向 | マスク 0xFF（レイヤー0時）、ビュー前方向 -Z |
| 向きがおかしい | ビュー前方向が +Z だった | 前方向を -Z に統一 |
| 原点から出ている | 行列から取った「カメラ位置」が誤り | カメラ位置を `_CameraPosition` で明示的に渡す |
| RT が彩度強く見える | RT がポスト処理を通っていない | ガンマの誤りではなくポストの有無。揃えるなら RT も同じポストを通すか簡易トーンをかける |
| ガンマがおかしい（RT が明るい） | sRGB テクスチャを GUI 表示する際にガンマ補正が二重にかかっていた可能性 | OutputTexture を R16G16B16A16_SFloat（リニア）に変更 |
